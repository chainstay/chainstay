Docker is changing the game for cloud-native applications. In order to accelerate application development, simplify continuous integration, and scale deployments, I do all of my work with Docker.

Advantages of Docker

  • Deploy Anywhere

    All of the major cloud providers are embracing Docker containers for how much they simplify application deployments. By building applications in Docker, I am able to deploy on whichever cloud provider you prefer (Amazon, Microsoft, or Google), for a lower cost than deploying virtual machines.

  • Reliability

    With a Dockerized application, it will run exactly the same way in development as it does in production. That means fewer bugs and less time spent configuring servers.

  • Lower costs

    Before containers, cloud servers had to spend enormous resources to maintain private virtual machines for each application. With containers, servers can be optimized to run containers instead of virtual machines, saving money on server infrastructure.

Services I offer

  • Continuous Integration (CI)

    When I make an improvement to your application, tests will automatically run to verify its integrity. That means fewer bugs and faster development, with greater confidence.

  • Continuous Deployment (CD)

    Once tests pass and new changes have been verified, they will be immediately deployed to a staging environment for further testing and review. That means that you will always be able to review the current state of the application rather than waiting until everything is finished.

  • Auto-scaling

    I build applications that monitor their own activity and provision more server resources when necessary. That means you won't have to pay for server resources you aren't using. Your app will also keep running even if you get a large spike in traffic.
